Class Hierarchy

  • object: The most base type
    • keybench.benchmark_component.BenchmarkComponent: Core component of all the keyphrase extractor system's components.
      • keybench.candidate_extractor.CandidateExtractorC: Component responsible of the term filtering.
        • keybench.default.n_gram_extractor.NGramExtractor: Component performing candidate terms extraction.
      • keybench.evaluator.EvaluatorC: Component responsible of the keyphrases evaluation, compared to reference keyphrases.
        • keybench.default.prf_evaluator.PRFEvaluator: Component performing keyphrases evaluation.
      • keybench.pre_processor.PreProcessorC: Component responsible of the document pre-processing.
        • keybench.default.stanford_pre_processor.StanfordPreProcessor: Component performing pre-processing of documents using the punkt sentence tokenizer, a regular expression word tokenizer and the stanford's tagger.
      • keybench.ranker.RankerC: Component responsible of the candidates' ranking.
        • keybench.default.tfidf_ranker.TFIDFRanker: Component performing candidate terms ranking based on their TF-IDF weight.
      • keybench.selector.SelectorC: Component responsible of the selection of the candidate keyphrases.
        • keybench.default.top_k_selector.TopKSelector: Component performing selecting the real keyphrases among the ranked candidate terms.
    • keybench.keyphrase_extractor.KeyphraseExtractor: Configuration of the keyphrase extraction system.
    • keybench.pre_processed_file.PreProcessedFile: Represents a file containing a title, an abstract and a body.
    • multiprocessing.process.Process: Process objects represent activity that is run in a separate process
      • keybench.keybench_worker.KeyBenchWorker: Process launching the keyphrase extraction workflow.
